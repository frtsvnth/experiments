# Sprint Review Show

Production-ready статическое веб-приложение для выбора очередности выступления команд на Sprint Review.

## Функциональность

- **7 игровых механик**: Колесо Фортуны, Плинко, Пинбол, Игровой Автомат, Ракетная Гонка, Хватайка, Тайные Карты.
- **Режим ведущего** — полноэкранное шоу для онлайн-встречи.
- **Админ-панель** — управление командами, drag-and-drop, скрытые сценарии очередности.
- **Управляемая случайность** — визуально случайный выбор с возможностью предопределить результат.
- **Честный случай** — настоящий random, когда сценарий не задан.
- **Сессии** — выбранные команды удаляются только из текущей сессии, мастер-список сохраняется.
- **PIN-защита админки** — доступ на уровне UI, не является реальной аутентификацией.
- **Экспорт/импорт JSON** — бэкап и перенос настроек.
- **Звуки и reduced-motion** — уважает `prefers-reduced-motion` (с ручным override в настройках).

## Локальный запуск

```bash
npm install
npm run dev
```

Откройте http://localhost:5173

## Сборка для GitHub Pages

```bash
npm run build
```

Содержимое папки `dist` загрузите в ветку `gh-pages` (или используйте `npm run deploy` при наличии `gh-pages` CLI).

## Управление

- **Пробел / Enter** — запуск выбора или переход к следующей команде.
- **F** — полноэкранный режим.
- **#admin** — переключение в админ-панель.

## Как включить scripted mode

1. Перейдите в Админ.
2. В разделе **Сценарий очередности** задайте:
   - **Полный порядок** — ID команд через запятую.
   - **Фиксированные позиции** — JSON вида `{"1":"team-id","4":"team-id"}`.
   - **Следующая команда** — команда, которая выйдет следующей принудительно.
3. Вернитесь в шоу и запустите выбор.

> ID команд можно посмотреть в отладочной панели админки или в экспортированном JSON.

## Ограничения безопасности

- PIN-код и админ-доступ реализованы исключительно на уровне UI.
- Все данные хранятся в `localStorage` на устройстве пользователя.
- Любой, у кого есть доступ к устройству или devtools, может изменить состояние.
- Это не backend-аутентификация и не защита данных.

## Архитектура

- `src/domain/` — чистые типы и сущности.
- `src/engine/` — `selectNextTeam`, единый движок выбора.
- `src/mechanics/` — адаптеры игровых механик, отделены от логики выбора.
- `src/state/` — React Context + reducer + localStorage persistence.
- `src/components/` — UI режима ведущего и админ-панели.
